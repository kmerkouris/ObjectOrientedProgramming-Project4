Αιτιολόγηση σχετικα με σχεδιαστηκές επιλογές:
-Η υλοποίηση όλων των κλάσεων σχεδιάστηκε σχεδόν κατά γράμμα, όσο μπορούσα, βάση της εκφώνησης.
-Η κλάση VehicleFactory και η κλάση controller θα μπορουσαν να υλοποιηθουν και ως namespaces.
Ο λόγος που η κλάση VehicleFactory δεν υλοποιήθηκε έτσι θα γίνει εμφανής σε λίγο.
-Η κλάση VehicleFactory έχει μια μέθοδο, την rangedRand, η οποία δεν είναι κατα ανάγκη συμπεριφορά
που χρειάζεται μόνο αυτή ή που είναι αναγκαία για την υλοποίησή της. Αρχικά αυτή η μέθοδος ήταν
κομμάτι μίας υλοποίησης της parent κλάσης της VehicleFactory, η οποία θεωρήσαμε οτι τελικά 
δεν χρειαζόταν την στιγμή που καταλάβαμε ότι θα χρειαζομασταν factory μόνο για τα αυτοκίνητα, άρα
απλά φέραμε τις μεθόδους που χρειαζόμασταν απο εκείνη την κλάση στην VehicleFactory και την διαγράψαμε.
Για να αποφύγουμε άσκοπες αλλαγές στον κώδικα, κρατήσαμε την υλοποίησή της ως κλάση.
-Οι κόμβοι δεν υλοποιήθηκαν ως κλάση, καθώς η λογική τους μπορεί απλα να υλοποιηθεί επιτρέποντας στα
αυτοκίνητα να μπαίνουν σε και να βγαίνουν απο κάθε τμήμα ανάλογα με την κατάστασή τους. Αν είναι έτοιμα
να φύγουν, φευγουν απο τον κόμβο στο τέλος του τμήματος που είναι. Αλλιώς πηγαίνουν στο επόμενο τμήμα.
Για τον ίδιο λόγο και επειδη οι κόμβοι που ελέγχονται για είσοδο και για έξοδο είναι ίδιου πλήθους (ΝSegs), 
η απαρίθμηση των κόμβων κάθε κατηγορίας αρχίζει απο το 0.

Παραδοχές:
-Το K αναφέρεται στον συνολικό αριθμό αυτοκινήτων που μπορεί να μπει απο τα επανδρωμένα διόδια στο τμήμα 
σε κάθε αλλαγή κατάστασης, και όχι στον αριθμό που μπορεί να μπεί απο κάθε διόδιο. Ομοίως για τα
ηλεκτρονικά διόδια με 2Κ.
-Κάθε είδοδος έχει από 1 μέχρι 8 διόδια κάθε είδους.
-Κάθε διόδιο ξεκινάει με 1 μέχρι 6 αυτοκίνητα.
-Κάθε αλλαγή κατάστασης, κάθε διόδιο δέχεται απο 0 έως 2 καινούρια αυτοκίνητα.

Σημειώσεις:
-Χρησιμοποιήθηκε το standard C++11
-Δεν έχει υλοποιηθεί κανενός είδους εντοπισμός λαθών ούτε στα command line arguments
ούτε στις τιμές που δίνονται απο το cin. Αν δεν δωθούν τιμές που είναι θετικοί ακέραιοι,
το πρόγραμμα θα εμφανίζει περίεργει συμπεριφορά ή μπορεί και να πετάξει exception.
-Προτίμησα την χρήση camelCase από snake_case για τα ονόματα των μελών και μεθόδων κάθε
κλάσης. Αν αυτό είναι πρόβλημα, διατείθομαι να προσφέρω εναλλακτικό project που τα ονόματα
είναι σε snake_case.
-Προτίμησα δυναμική καταχώρηση μνήμης γιατί μου δίνει παραπάνω έλεγχο στο πρόγραμμα, εκτός
απο περιπτώσεις container όπως το vector
